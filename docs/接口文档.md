# TestMasterProject 接口文档

## 基础信息
- **后端地址**：http://localhost:5001
- **前端地址**：http://localhost:3000
- **统一返回格式**：
  ```json
  {
    "code": 0,          // 0表示成功，非0表示失败
    "msg": "成功",       // 提示信息
    "data": {}          // 返回数据
  }
  ```
- **认证方式**：请求头携带 `Authorization: Bearer <token>`

---

## 一、认证相关接口

### 1. 用户注册
**接口地址**：`POST /api/register`
**无需认证**
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名，3-20位 |
| email | string | 是 | 邮箱，符合邮箱格式 |
| password | string | 是 | 密码，6-32位 |

**请求示例**：
```json
{
  "username": "testuser",
  "email": "test@test.com",
  "password": "123456"
}
```

**返回示例**：
```json
{
  "code": 0,
  "msg": "注册成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "testuser",
      "email": "test@test.com"
    }
  }
}
```

---

### 2. 用户登录
**接口地址**：`POST /api/login`
**无需认证**
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**请求示例**：
```json
{
  "username": "testuser",
  "password": "123456"
}
```

**返回示例**：
```json
{
  "code": 0,
  "msg": "登录成功",
  "data": {

    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
```

---

### 3. 修改密码
**接口地址**：`POST /api/change-password`
**需要普通用户认证**
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| old_password | string | 是 | 旧密码 |
| new_password | string | 是 | 新密码，6-32位 |

**请求示例**：
```json
{
  "old_password": "123456",
  "new_password": "654321"
}
```

**返回示例**：
```json
{
  "code": 0,
  "msg": "密码修改成功"
}
```

---

### 4. 刷新token
**接口地址**：`POST /api/refresh`
**需要refresh_token认证**
**请求参数**：无
**返回示例**：
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

---

### 5. 获取用户信息
**接口地址**：`GET /api/profile`
**需要普通用户认证**
**请求参数**：无
**返回示例**：
```json
{
  "user": {
    "id": 1,
    "username": "testuser",
    "email": "test@test.com",
    "created_at": "2024-03-15T12:00:00"
  }

```

---

### 6. 退出登录
**接口地址**：`POST /api/logout`
**无需认证（客户端自行清除token即可）**
**返回示例**：
```json
{
  "message": "Logout successful"
}
```

## 三、用户管理接口

### 1. 获取用户列表
**接口地址**：`GET /api/admin/users`
**需要管理员认证**
**请求参数**（Query参数）：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认1 |
| size | int | 否 | 每页数量，默认10 |
| keyword | string | 否 | 搜索关键词（用户名/邮箱） |
| status | string | 否 | 状态筛选：active/disabled |

**返回示例**：
```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "id": 1,
        "username": "testuser",
        "email": "test@test.com",
        "level": 2,
        "score": 100,
        "status": "active",
        "is_admin": false,
        "registerTime": "2024-03-15 12:00:00"
      }
    ],
    "total": 100,
    "page": 1,
    "size": 10
  }
}
```

---

### 2. 新增用户
**接口地址**：`POST /api/admin/users`
**需要管理员认证**
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| email | string | 是 | 邮箱 |
| password | string | 是 | 密码 |
| level | int | 否 | 等级，默认1 |
| score | int | 否 | 积分，默认0 |
| is_active | boolean | 否 | 是否启用，默认true |
| is_admin | boolean | 否 | 是否为管理员，默认false |

**请求示例**：
```json
{
  "username": "newuser",
  "email": "new@test.com",
  "password": "123456",
  "level": 1,
  "is_active": true
}
```

---

### 3. 编辑用户
**接口地址**：`PUT /api/admin/users/:user_id`
**需要管理员认证**
**路径参数**：`user_id` - 用户ID
**请求参数**：同新增用户（密码可选，不填则不修改）

---

### 4. 删除用户
**接口地址**：`DELETE /api/admin/users/:user_id`
**需要管理员认证**
**路径参数**：`user_id` - 用户ID
**返回示例**：
```json
{
  "code": 0,
  "msg": "用户删除成功"
}
```

---

### 5. 切换用户状态
**接口地址**：`PUT /api/admin/users/:user_id/toggle-status`
**需要管理员认证**
**路径参数**：`user_id` - 用户ID
**返回示例**：
```json
{
  "code": 0,
  "msg": "状态切换成功"
}
```

---

### 6. 设置/取消管理员权限
**接口地址**：`PUT /api/admin/users/:user_id/admin`
**需要超级管理员认证**
**路径参数**：`user_id` - 用户ID
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| is_admin | boolean | 是 | true=设置为管理员，false=取消管理员 |

**请求示例**：
```json
{
  "is_admin": true
}
```

---

### 7. 重置用户密码
**接口地址**：`PUT /api/admin/users/:user_id/reset-password`
**需要管理员认证**
**路径参数**：`user_id` - 用户ID
**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| new_password | string | 是 | 新密码，6-32位 |

**请求示例**：
```json
{
  "new_password": "654321"
}
```

## 错误码说明
| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误 |
| 401 | 未登录或登录已过期 |
| 403 | 没有权限访问 |
| 404 | 请求的资源不存在 |
| 409 | 资源已存在（如用户名/邮箱重复） |
| 500 | 服务器内部错误 |
