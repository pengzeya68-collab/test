TestMaster Desktop 1.0.0 本地试用说明

启动方式
1. 双击“00-点这里启动-TestMaster桌面版.bat”。
2. 启动器会自动检查并启动 5001 后端，然后打开桌面客户端。
3. 首次打开浏览器录制器可能需要数秒，Chromium 已包含在安装包中，无需安装驱动。

本地试用账号
管理员：admin / admin123
自动化测试账号：auto_test / 123456

主要入口
- UI 自动化用例：录制、编辑、断言、调试和运行单条业务流程。
- 回归套件：组合多个用例，配置多行 JSON 测试数据并顺序执行。
- 运行记录：查看步骤结果、失败截图，并保存 Playwright Trace。

日志
- 后端标准日志：logs\desktop-backend.log
- 后端错误日志：logs\desktop-backend-error.log

企业部署
桌面客户端是浏览器执行代理，用户、用例、环境、运行记录和制品由 TestMaster 后端统一管理。
正式多人使用时应部署独立后端与数据库，在登录页配置企业服务地址，不应共享本地 SQLite 数据库或固定演示账号。


登录态运行前检查
- 单用例和回归套件在正式创建运行记录前，会使用内置 Chromium 访问首个业务页面。
- Cookie 全部过期、跳回登录/验证码地址、出现密码输入框或服务 5xx 时会提前阻断。
- 点击“管理登录态”可查看更新时间、手动验活或删除废弃状态。
- 测试环境和登录态选择会真实传入执行器，并记录到运行记录中。
