## 批量导入接口测试用例

### 任务
将项目所有后端 API 整理为接口测试用例，按模块分文件夹导入到接口测试模块

### 完成情况
- 新建 9 个文件夹，导入 77 个接口测试用例
- 同时更新默认环境变量，新增 `base_url` 和 `refresh_token` 变量
- 脚本位于 `scripts/seed_interface_test_cases.py`（主数据）和 `scripts/run_seed_cases.py`（执行入口）

### 文件夹结构
1. **01 认证模块 (Auth)** - 9 个用例（注册/登录/Profile/修改密码/验证码/刷新Token等）
2. **02 习题模块 (Exercises)** - 11 个用例（CRUD/提交/执行SQL等）
3. **03 学习路径 (Learning Paths)** - 8 个用例（CRUD/习题管理等）
4. **04 考试模块 (Exam)** - 8 个用例（列表/详情/开始/提交/结果/智能生成等）
5. **05 面试题模块 (Interview)** - 5 个用例（列表/详情/收藏等）
6. **06 社区模块 (Community)** - 14 个用例（帖子/评论/点赞/收藏等）
7. **07 技能统计 (Skills)** - 4 个用例（雷达图/统计/进度/维度）
8. **08 接口测试管理 (Interface Test)** - 14 个用例（文件夹/用例/环境的完整CRUD）
9. **09 管理员模块 (Admin)** - 4 个用例（备份/用户管理）

### 环境配置
- 默认环境「本地开发环境」base_url=http://localhost:3000
- 变量：base_url / api_prefix(/api) / token / refresh_token
- URL 格式：`{{base_url}}{{api_prefix}}/xxx`
- 所有需要 Token 的用例 Header 已设为 `Authorization: Bearer {{token}}`

### 使用说明
1. 前端登录 admin 账号
2. 进入接口测试，选择「本地开发环境」
3. 先运行「用户登录」用例，复制 access_token
4. 在环境管理中将 token 变量值更新为 access_token
5. 其他接口即可直接测试
