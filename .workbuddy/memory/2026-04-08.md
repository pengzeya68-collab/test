# 2026-04-08 工作日志

## 修复：用例编辑保存后数据为空（致命Bug）

### 根因：前后端字段名完全不匹配 + 编辑时没获取完整数据

**涉及文件**：`backend/api/auto_test.py` + `frontend/src/views/CaseEditorDrawer.vue`

**问题1 - 字段名不匹配**：
- 前端发送 `group_id`，后端只认 `folder_id`
- 前端发送 `payload`，后端只认 `body`
- `get_cases` / `get_case` API 不返回前端期望的 `payload` 和 `group_id` 字段

**问题2 - 编辑时用列表数据而非详情数据**：
- `CaseEditorDrawer` 打开时直接用列表行的摘要数据初始化表单
- 列表 API 不返回 `extractors`、`assert_rules` 等字段
- 没有在编辑打开时调用 `GET /api/auto-test/cases/{id}` 获取完整数据

**修复内容**：
1. 后端 `get_cases` / `get_case`：在返回中额外添加 `payload`（=body）和 `group_id`（=folder_id）
2. 后端 `create_case` / `update_case`：接收 `group_id` 映射到 `folder_id`，接收 `payload` 映射到 `body`
3. 前端 `CaseEditorDrawer`：新增 `watch(modelValue)` 当抽屉打开时，如果是编辑模式则强制调用详情 API 获取完整数据
