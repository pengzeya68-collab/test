## 面试会话 bug 修复 (22:08)
### 问题1：500 错误 - `NOT NULL constraint failed: interview_sessions.title`
- 根因：`InterviewSession` 插入时没传 `title`，数据库约束失败
- 修复：新建会话时自动生成 `title=f"{position}-{type}模拟面试"`

### 问题2："继续面试"变成新建会话（完整修复 22:23）
- 根因 A：`form` 没有 `session_id` 字段，前端无法告诉后端要继续哪个会话
- 根因 B：复用时 `question_list` 只有 question_id，没有完整题目信息
- 根因 C：`fetchSession` 用 `GET /sessions/{id}` 但该接口不返回 questions
- 根因 D：组件挂载时不检查 URL query 参数，无法自动触发加载

### 修复内容
1. **前端 `form`**：加 `session_id: null` 字段
2. **前端 `fetchSession`**：改用 `POST /sessions/batch` + `session_id` 获取完整 question_list
3. **前端 `onMounted`**：检测 URL `?session_id=X` 参数，自动调用 `fetchSession()`
4. **后端复用逻辑**：从 submissions 恢复时，先批量查出完整题目信息再构建 question_list
