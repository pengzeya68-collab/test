# 2026-04-07 工作日志

## 修复：自动化测试接口执行报错 + 添加场景报错

### Bug 1: 执行接口报错 (致命)
**文件**: `backend/api/auto_test.py` - `run_scenario` 和 `execute_single_step` 函数

**根因**: 
1. `run_scenario` 函数第 865 行：`step.api` 属性不存在（模型字段名是 `step.api_case`），导致 `AttributeError`
2. `execute_single_step` 函数第 971 行：`start_time` 变量从未定义就使用了，导致 `NameError`
3. `execute_single_step` 第 973 行：`success = 200 <= response.status_code < 300` 只拦截 4xx/5xx 但放过了 3xx

**修复**:
- `step.api` → `step.api_case` (第865行)
- 添加 `step_start_time = datetime.utcnow()` 替代不存在的 `start_time`
- 改为 `success = 200 <= response.status_code < 400` 并增加 `if response.status_code >= 400` 的提前拦截

### Bug 2: 添加场景报错
**文件**: `backend/api/auto_test.py` - `create_scenario` 函数

**根因**: `case_ids=json.dumps(data.get('case_ids', []))` 中，如果 `case_ids` 为 `None`（前端未传），`data.get('case_ids', [])` 返回 `None` 而非 `[]`（因为 JSON `null` 会覆盖默认值）。另外缺少异常处理。

**修复**:
- 提取 `case_ids = data.get('case_ids', [])` 到独立变量，确保有值
- 添加 try/except 包裹 db 操作，失败时 rollback

### Bug 3: 数据驱动执行是空壳
**文件**: `backend/api/auto_test.py` - `run_scenario_data_driven` 函数

**根因**: 原实现是占位代码，直接返回空结果，没有实际执行任何步骤

**修复**:
- 完整实现数据驱动循环：逐行遍历 rows，每行注入变量到上下文
- 复用 `execute_single_step` 执行步骤
- 支持前端内联传 `data_matrix` 或从数据库读取
- 每步失败后 break 跳过后续步骤
