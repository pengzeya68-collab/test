## 修复习题库前端展示Bug
发现两个明显的前端展示问题并修复:

### Bug 1: 习题库列表显示为空
**原因**: 前端 `Exercises.vue` 默认 `currentStage = 1`，筛选逻辑只显示 `stage === 1` 的习题。但数据库中897道题的 `stage` 全部为 NULL，导致全部被过滤。

**修复** (`frontend/src/views/Exercises.vue`):
```js
// 修改前
result = result.filter(item => item.stage === currentStage.value)
// 修改后
result = result.filter(item => item.stage === currentStage.value || item.stage === null || item.stage === undefined)
```

### Bug 2: 选择题渲染成问答形式(文本输入框)
**原因**: `ExerciseDetail.vue` 的 `isChoiceType` 计算属性只检查了 `choice/multiple_choice/true_false`，没有包含 `single_choice`。数据库中322道单选题因此落入 `v-else` 分支，显示为 textarea。

**修复** (`frontend/src/views/ExerciseDetail.vue`):
```js
// 修改前
return t === 'choice' || t === 'multiple_choice' || t === 'true_false'
// 修改后
return t === 'choice' || t === 'single_choice' || t === 'multiple_choice' || t === 'true_false'
```

### 额外修复
- `Exercises.vue` 中题型筛选 `typeMap` 的 `choice` 数组也补上了 `single_choice`
- `getExerciseTypeTagType` 和 `getExerciseTypeText` 函数增加了 `single_choice` 的映射显示

## RBAC 权限系统实现（完整完成）

### 实现方案
采用**基础 RBAC**：3 种角色 + 权限表，暂不需要前端管理界面。

### 完成内容

#### 1. 数据库模型 (`fastapi_backend/models/models.py`)
- 新增 `Role` 模型（角色表）
- 新增 `Permission` 模型（权限表）
- 新增 `RolePermissionMapping` 模型（角色-权限关联表）
- `User` 模型新增 `role_id` 外键字段
- `User` 模型新增 `role_obj` 关联和 `permissions` 属性
- `User.role` 属性优先从 `role_obj.name` 获取，回退到 `is_admin` 字段（向后兼容）

#### 2. 权限检查 (`fastapi_backend/deps/auth.py`)
- 新增 `require_permission(*required_permissions)` 依赖函数
- 支持通配符 `*` （admin 拥有所有权限）
- 支持模块级通配符（如 `exercise:*`）
- `require_admin` 等现有函数**保持不变**（向后兼容）

#### 3. Pydantic Schemas (`fastapi_backend/schemas/rbac.py`)
- `RoleSchema` / `RoleCreate` / `RoleUpdate`
- `PermissionSchema` / `PermissionCreate`
- `PermissionAssign` / `UserRoleAssign`

#### 4. RBAC 管理路由 (`fastapi_backend/routers/rbac.py`)
- `GET /api/v1/admin/rbac/roles` — 列出所有角色
- `POST /api/v1/admin/rbac/roles` — 创建角色
- `PUT /api/v1/admin/rbac/roles/{id}` — 更新角色
- `DELETE /api/v1/admin/rbac/roles/{id}` — 删除角色
- `GET /api/v1/admin/rbac/permissions` — 列出所有权限
- `POST /api/v1/admin/rbac/permissions` — 创建权限
- `GET /api/v1/admin/rbac/roles/{id}/permissions` — 查看角色权限
- `POST /api/v1/admin/rbac/roles/{id}/permissions` — 分配权限给角色
- `PUT /api/v1/admin/rbac/users/{id}/role` — 为用户分配角色

#### 5. 路由注册 (`fastapi_backend/main.py`)
- 导入 `rbac_router`
- 注册到 `admin_router_group`

#### 6. 数据库迁移
- 创建 Alembic 迁移文件 `alembic/versions/3a62ae184c47_add_rbac_tables.py`
- **问题**: Alembic `alembic.exe` 初次未安装，后安装成功但 `alembic upgrade` 因数据库文件锁报错
- **解决**: 直接用 SQL 手动创建表（`roles` / `permissions` / `role_permissions`），然后 `alembic stamp head` 标记迁移为已执行

#### 7. 种子数据 (`fastapi_backend/scripts/seed_rbac.py`)

### 经验教训

1. **Alembic 路径问题**: `DATABASE_URL` 使用相对路径时，需从项目根目录运行 `alembic`，否则找不到 `instance/` 目录
2. **Alembic stamp 不会建表**: `alembic stamp` 只标记版本号，不执行实际迁移 SQL；需先手动建表再 stamp
3. **SQLite 外键约束**: SQLite 不支持 `ALTER TABLE DROP CONSTRAINT`，需重建表
4. **路径计算**: Python 脚本中用 `Path(__file__).resolve().parent.parent.parent` 计算项目根目录（3 层）
5. **种子脚本不要依赖项目配置**: 独立版种子脚本直接连 SQLite，避免因 `sys.path` 问题导致导入失败
