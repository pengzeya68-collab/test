## 双重序列化 Bug 修复 (17:58)
### 问题根因
`send_request` 接口（autotest_execution.py 第 204-209 行）：前端 body 是 JSON 字符串 `'{"key": "value"}'`，直接传给 `requests.post(..., json=string)`，requests 底层把它再序列化一次 → 目标服务器收到 `'"{\\"key\\": \\"value\\"}"'` → 400 错误

### 修复内容
在 `body_type == "json"` 时，若 `processed_body` 是字符串，先 `json.loads()` 反序列化为 dict，再传给 `requests.json=`；JSON 解析失败时 `raise HTTPException(400, detail="请求体 JSON 格式校验失败，请检查语法")`
### 文件
- `fastapi_backend/routers/autotest_execution.py` 第 204-227 行
- 需要重启后端服务
